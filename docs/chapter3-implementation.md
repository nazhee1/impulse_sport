# Глава 3. Реализация и тестирование

## 3.1 Выбор средств разработки

Для демонстрационного прототипа интернет-магазина спортивных товаров были использованы HTML, CSS и JavaScript. Такой набор технологий позволяет быстро реализовать пользовательский интерфейс, показать структуру системы и продемонстрировать основные сценарии работы приложения.

HTML используется для описания структуры страниц, CSS — для оформления и адаптивности интерфейса, JavaScript — для интерактивного поведения, включая поиск, фильтрацию и работу корзины.

Для полной реализации проекта в дальнейшем может применяться серверная часть на Node.js с Express или Python с Django, а для хранения данных — PostgreSQL.

## 3.2 Реализация пользовательского интерфейса

В ходе разработки был создан демонстрационный интерфейс интернет-магазина `SportStore`. В прототипе реализованы следующие элементы:

1. главная страница с описанием проекта;
2. блок категорий спортивных товаров;
3. каталог товаров;
4. поиск по названию и описанию;
5. фильтрация по категории;
6. фильтрация по максимальной цене;
7. добавление товаров в корзину;
8. расчет общего количества товаров и суммы заказа;
9. демонстрационный блок административной аналитики.

Интерфейс выполнен в адаптивной форме, поэтому его удобно просматривать как на компьютере, так и на мобильном устройстве. Такой подход важен для интернет-магазина, так как значительная часть пользователей совершает покупки со смартфонов.

## 3.3 Логика работы прототипа

В клиентской части реализован массив тестовых товаров, каждый из которых содержит идентификатор, название, категорию, описание, цену и рейтинг.

При изменении параметров поиска или фильтрации происходит повторный отбор товаров по следующим условиям:

- совпадение поискового запроса;
- принадлежность выбранной категории;
- соответствие максимальной цене.

При добавлении товара в корзину система увеличивает количество единиц выбранной позиции, затем пересчитывает итоговые показатели:

- общее количество товаров;
- суммарную стоимость заказа.

Хотя текущая версия работает как демонстрационный прототип и не сохраняет заказы в реальную базу данных, логика взаимодействия уже отражает структуру будущего полнофункционального решения.

## 3.4 Тестирование

Для оценки корректности работы были определены основные пользовательские сценарии:

1. открытие главной страницы;
2. переход к каталогу;
3. поиск товара по ключевому слову;
4. фильтрация по категории;
5. фильтрация по цене;
6. добавление товара в корзину;
7. расчет итоговой суммы.

Результаты тестирования показали:

- каталог отображается корректно;
- поиск отбирает соответствующие товары;
- фильтрация по категории работает корректно;
- фильтрация по цене ограничивает список товаров;
- корзина обновляется после добавления товара;
- итоговая сумма рассчитывается правильно.

Следовательно, основные сценарии демонстрационного прототипа работают в соответствии с поставленной задачей.

## 3.5 Преимущества реализованного решения

Разработанный проект имеет следующие преимущества:

1. понятная структура интерфейса;
2. наличие ключевых функций интернет-магазина;
3. логичная модель данных;
4. возможность дальнейшего расширения;
5. практическая направленность.

Даже в формате учебного прототипа система уже демонстрирует базовые принципы проектирования веб-приложений и может быть развита до полноценного программного продукта.

## 3.6 Перспективы развития

В дальнейшем проект можно расширить в следующих направлениях:

- реализовать серверную часть;
- подключить реальную базу данных;
- добавить регистрацию и авторизацию;
- реализовать хранение истории заказов;
- внедрить онлайн-оплату;

- добавить отзывы и систему рекомендаций;
- подключить уведомления по электронной почте или в мессенджерах.
