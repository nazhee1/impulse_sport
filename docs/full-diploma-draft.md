## Введение

Выпускная квалификационная работа посвящена разработке интернет-магазина спортивных товаров. Выбор данной темы обусловлен развитием электронной коммерции и ростом спроса на удобные онлайн-сервисы для выбора и приобретения товаров. В сфере спортивных товаров особенно важны структурированный каталог, удобная фильтрация, понятная карточка товара и быстрое оформление заказа.

Цель работы заключается в разработке информационной системы интернет-магазина спортивных товаров, которая позволяет автоматизировать основные процессы онлайн-продаж.

Для достижения поставленной цели были определены следующие задачи:

1. Проанализировать предметную область.
2. Сформировать требования к системе.
3. Спроектировать архитектуру решения.
4. Разработать структуру базы данных.
5. Подготовить API взаимодействия.
6. Создать пользовательский прототип.
7. Подготовить материалы для защиты проекта.

Объект исследования — процесс онлайн-продажи спортивных товаров.

Предмет исследования — методы и средства проектирования веб-приложений для автоматизации интернет-продаж.

Практическая значимость проекта состоит в том, что разработанная система может выступать прототипом для реального магазина спортивных товаров.

## Глава 1. Анализ предметной области

Интернет-магазин спортивных товаров должен обеспечивать пользователю быстрый и удобный доступ к каталогу продукции. Покупателю важно видеть категории товаров, цены, описания и возможность фильтрации. Для владельца бизнеса важны обработка заказов, управление каталогом и хранение данных.

В ходе анализа были выявлены ключевые требования:

- наличие каталога товаров;
- поиск и фильтрация;
- карточка товара;
- корзина;
- оформление заказа;
- административные функции.

Проект относится к информационным системам, так как работает не только с отображением страниц, но и с хранением и обработкой данных о пользователях, товарах и заказах.

## Глава 2. Проектирование системы

В качестве архитектурного решения выбрана клиент-серверная модель. В ней пользователь работает через веб-интерфейс, клиентская часть обменивается данными с сервером по REST API, а сервер взаимодействует с базой данных.

Основные сущности базы данных:

- пользователи;
- роли;
- категории;
- бренды;
- товары;
- изображения товаров;
- корзины;
- позиции корзины;
- заказы;
- позиции заказа;
- платежи;
- отзывы.

Такое проектирование позволяет организовать целостное хранение данных и подготовить систему к дальнейшему масштабированию.

## Глава 3. Реализация и тестирование

Для демонстрации проекта был реализован прототип пользовательского интерфейса с помощью HTML, CSS и JavaScript. Прототип показывает основные функции:

- вывод каталога;
- поиск;
- фильтрацию по категории;
- фильтрацию по цене;
- добавление товара в корзину;
- расчет итоговой суммы.

Проведенное тестирование показало корректность работы базовых сценариев.

## Заключение

По итогам работы была разработана концепция интернет-магазина спортивных товаров, спроектирована архитектура системы, структура базы данных и создан демонстрационный прототип. Цель работы достигнута, а сам проект может использоваться как база для дальнейшей полноценной реализации.
