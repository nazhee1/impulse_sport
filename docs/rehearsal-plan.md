# План подготовки к защите

Этот файл нужен, чтобы быстро подготовиться и не перегрузить себя в последний день.

## Если у вас есть 1 день

1. Прочитайте [full-diploma-draft.md](full-diploma-draft.md), чтобы понимать общую логику проекта.
2. Выучите [defense-speech.md](defense-speech.md) не слово в слово, а по смысловым блокам.
3. Просмотрите [presentation-10-slides.md](presentation-10-slides.md) и соберите слайды.
4. Пройдитесь по [defense-show-script.md](defense-show-script.md) и один раз покажите проект вслух.
5. Повторите [quick-cheatsheet.md](quick-cheatsheet.md).
6. Прочитайте [defense-qa.md](defense-qa.md) и [hard-questions.md](hard-questions.md).

## Если у вас есть 2-3 часа

1. Выучите тему, цель, задачи, объект и предмет.
2. Поймите архитектуру: frontend, backend, API, база данных.
3. Запомните основные сущности базы данных.
4. Повторите, что реализовано в прототипе, а что пока нет.
5. Один раз проговорите защитную речь вслух.
6. Один раз откройте прототип и покажите сценарий от главной страницы до корзины.

## Что нужно знать наизусть

- тема проекта;
- актуальность;
- цель;
- задачи;
- объект;
- предмет;
- архитектура;
- роли пользователей;
- основные функции;
- что реализовано;
- практическая значимость;
- перспективы развития.

## Самая безопасная схема ответа

Если задают вопрос, отвечайте по схеме:

1. короткое определение;
2. зачем это нужно в проекте;
3. как это реализовано или предусмотрено у вас.

Пример:

REST API — это способ обмена данными между клиентом и сервером. В моем проекте он нужен для передачи данных о товарах, корзине и заказах. В текущей версии он описан на уровне проектирования и подготовлен для дальнейшей реализации backend-части.

## Что не нужно делать на защите

- не говорить слишком быстро;
- не обещать функции, которых нет;
- не спорить с комиссией;
- не уходить в лишние детали, если вопрос простой;
- не говорить «я не знаю вообще».

## Что говорить, если волнуетесь

Если вы забыли формулировку, возвращайтесь к простой конструкции:

Мой проект — это информационная система интернет-магазина спортивных товаров. В нем я спроектировал архитектуру, базу данных, API и создал рабочий интерфейсный прототип. Основная цель проекта — автоматизация процесса онлайн-продаж.
