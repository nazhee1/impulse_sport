# План тестирования

Этот документ можно использовать в разделе тестирования и как ответ на вопрос о проверке проекта.

## Цель тестирования

Проверить корректность работы основных пользовательских сценариев интернет-магазина спортивных товаров.

## Основные тестовые сценарии

| ID | Сценарий | Действие | Ожидаемый результат |
|---|---|---|---|
| T1 | Открытие главной страницы | Открыть прототип | Страница загружается без ошибок |
| T2 | Отображение каталога | Перейти к каталогу | Список товаров отображается корректно |
| T3 | Поиск товара | Ввести запрос `кроссовки` | Показываются только подходящие товары |
| T4 | Фильтр по категории | Выбрать категорию `Фитнес` | Остаются товары из выбранной категории |
| T5 | Фильтр по цене | Уменьшить максимальную цену | Отображаются товары не дороже выбранного значения |
| T6 | Добавление в корзину | Нажать `В корзину` | Товар появляется в корзине |
| T7 | Подсчет количества | Добавить несколько товаров | Счетчик корзины обновляется корректно |
| T8 | Подсчет суммы | Добавить товары разной цены | Итоговая сумма считается правильно |
| T9 | Пустой результат поиска | Ввести несуществующий запрос | Показывается сообщение, что товары не найдены |
| T10 | Адаптивность | Открыть прототип на узком экране | Интерфейс перестраивается без критических ошибок |

## Итоги тестирования

По результатам проверки основные сценарии демонстрационного прототипа выполняются корректно. Это подтверждает работоспособность реализованных функций и пригодность прототипа для демонстрации на защите.

## Что можно сказать комиссии

Тестирование проводилось по основным пользовательским сценариям. Проверялись корректность отображения каталога, работа поиска и фильтрации, добавление товаров в корзину, пересчет итоговой суммы и адаптивность интерфейса.
