# Сценарий демонстрации проекта

Этот сценарий нужен, чтобы на защите вы не думали, что нажимать и в каком порядке показывать проект.

## Порядок показа

### 1. Главная страница

Скажите:

Перед вами главная страница интернет-магазина спортивных товаров. На ней показаны ключевая идея проекта, основные преимущества и переход к каталогу товаров.

### 2. Категории товаров

Скажите:

Ниже представлены основные категории ассортимента: товары для бега, фитнеса и командных видов спорта. Такое деление помогает пользователю быстрее ориентироваться в каталоге.

### 3. Каталог

Прокрутите до каталога и скажите:

В каталоге пользователь может просматривать товары, искать их по названию и фильтровать по категории и цене.

### 4. Поиск

Введите в поиск слово `кроссовки` и скажите:

Здесь показана функция поиска по названию и описанию товара. Это упрощает навигацию по каталогу.

### 5. Фильтрация по категории

Выберите категорию `Фитнес` и скажите:

Дополнительно реализована фильтрация по категориям. Это позволяет быстро отобрать нужную группу товаров.

### 6. Фильтрация по цене

Подвигайте ползунок максимальной цены и скажите:

Также реализована фильтрация по стоимости, что особенно важно для интернет-магазина при сравнении товаров.

### 7. Корзина

Нажмите `В корзину` у любого товара и скажите:

После добавления товара открывается корзина. В ней пользователь видит выбранные позиции, количество товаров и итоговую стоимость заказа.

### 8. Блок архитектуры

Покажите блок `Клиент-серверная модель` и скажите:

В проекте заложена клиент-серверная архитектура. Пользователь работает с интерфейсом, запросы обрабатываются сервером через API, а данные хранятся в базе данных.

### 9. Блок администрирования

Прокрутите к административной аналитике и скажите:

Здесь показана идея административной части системы, где можно контролировать заказы, остатки и основные показатели магазина.

### 10. Завершение показа

Скажите:

Таким образом, проект демонстрирует ключевые функции интернет-магазина и может быть расширен до полноценного веб-приложения с backend, авторизацией и реальной базой данных.

## Что не забыть сказать комиссии

- проект имеет практическую направленность;
- в нем есть роли пользователей;
- продумана структура базы данных;
- описан API;
- прототип демонстрирует основные пользовательские сценарии;
- система может быть расширена без изменения базовой архитектуры.
